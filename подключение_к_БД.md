# Подключение к БД

## Что уже сделано

- Приложение теперь умеет работать не только с локальной SQLite, но и с PostgreSQL/MySQL/Firebird через JDBC.
- Добавлены:
  - `data/db.properties.example` для шаблона настроек;
  - `setup.sh`, `run.sh`, `run_test.sh` для Linux/macOS;
  - `setup.bat`, `run.bat` для Windows;
  - `src/util/DatabaseCli.java` для `probe` и `seed`.

## Куда вписывать параметры БД

Скопируйте шаблон:

```bash
cp data/db.properties.example data/db.properties
```

Дальше редактируйте `data/db.properties` по тем же строкам, что и в шаблоне:

- Строка `4`: `db.type`
  - `sqlite` для локальной базы
  - `postgresql` для PostgreSQL сервера
  - `mysql` для MySQL сервера
  - `firebird` для Ред БД / Firebird
  - `custom` для другой JDBC-СУБД
- Строка `7`: `db.sqlitePath`
  - путь к локальному `.db` файлу
- Строки `10-15`: PostgreSQL-сервер
  - `db.host`
  - `db.port`
  - `db.name`
  - `db.username`
  - `db.password`
  - `db.sslMode`
- Строки ниже блока PostgreSQL можно использовать и для MySQL
  - `db.host=localhost`
  - `db.port=3306`
  - `db.name=todolist`
  - `db.username=root`
  - `db.password=...`
- Последние строки: `db.jdbcUrl` и `db.driverClass`
  - используются, если будете подключать другую JDBC-БД

Для Ред БД / Firebird можно использовать ваши данные:

```properties
db.type=firebird
db.host=192.168.1.6
db.port=3050
db.name=emp
db.username=user113
db.password=L@b$-113
```

Для PostgreSQL получится так:

```properties
db.type=postgresql
db.host=3e24e-rw.db.pub.dbaas.postgrespro.ru
db.port=5432
db.name=dbstud
db.username=chernov_rd
db.password=ВАШ_ПАРОЛЬ
db.sslMode=require
```

Для MySQL пример такой:

```properties
db.type=mysql
db.host=localhost
db.port=3306
db.name=todolist
db.username=root
db.password=ВАШ_ПАРОЛЬ
```

Важный плюс: после этой доработки для переключения между `postgresql`, `mysql` и `firebird` тебе достаточно менять только строку `db.type`, потому что в `data/db.properties` теперь можно хранить отдельные профили:

- `db.postgresql.*`
- `db.mysql.*`
- `db.firebird.*`

Пример:

```properties
db.type=firebird
```

Потом для PostgreSQL:

```properties
db.type=postgresql
```

И больше ничего менять не нужно, если профили уже заполнены.

## Какие файлы менять, если подключаете другую СУБД

- `src/repository/DatabaseConfig.java`
  - чтение конфига из `data/db.properties` и ENV
  - проверка обязательных полей
  - формирование JDBC URL и выбор драйвера для `sqlite/postgresql/mysql/firebird`
- `src/repository/ConnectionFactory.java`
  - единая фабрика JDBC-соединений
- `src/repository/SqlDialect.java`
  - различия `UPSERT`, `INSERT IGNORE`, автоинкремента и `ALTER TABLE`
- `src/repository/TaskDao.java`
  - единый DAO-интерфейс задач
- `src/repository/DatabaseManager.java`
  - старт инициализации БД: метод `initializeDatabase()`
  - DDL для таблиц
  - SQL для начального наполнения
  - миграции схемы
  - UPSERT задач
  - пользователи, коды восстановления и привязка задач к владельцу
- `src/repository/ConnectionFactory.java`
  - вызов `open(DatabaseConfig config)` делегирует открытие JDBC-соединения дальше
- `src/repository/DatabaseConfig.java`
  - метод `openConnection()` делает фактический вызов `DriverManager.getConnection(...)`
- `src/util/DatabaseCli.java`
  - SQL-проверка `CREATE TABLE -> INSERT -> SELECT -> DROP`
  - массовая генерация задач

Готовые схемы под курсовое задание:

- `sql/postgresql_schema.sql`
- `sql/mysql_schema.sql`
- `sql/firebird_schema.sql`

Если новая БД требует другой синтаксис SQL, править нужно в первую очередь `DatabaseManager.java` и `DatabaseCli.java`.

## Как проверить подключение

Сейчас для ручной проверки используется `DatabaseCli`.

Сценарий проверки такой:

1. Собирает проект.
2. Подключается к БД через JDBC.
3. Создаёт временную таблицу `codex_connection_probe`.
4. Вставляет строку.
5. Делает `SELECT`.
6. Удаляет таблицу.

Сначала соберите проект:

```bash
./run.sh --build-only
```

или на Windows:

```bat
run.bat --build-only
```

Потом запустите сам `probe`:

macOS / Linux:

```bash
java -cp "bin:lib/*" util.DatabaseCli probe
```

Windows:

```bat
java -cp "bin;lib/*" util.DatabaseCli probe
```

## Как быстро заполнить базу тестовыми задачами

Для массового наполнения используется та же `DatabaseCli`.

macOS / Linux:

```bash
java -cp "bin:lib/*" util.DatabaseCli seed 1000
```

Windows:

```bat
java -cp "bin;lib/*" util.DatabaseCli seed 1000
```

Команда создаёт указанное число разных задач с разными датами, приоритетами, категориями и тегами.

## Если будут ошибки

### Ошибка DNS / host not found

Причина:

- нет доступа к сети;
- сервер недоступен из вашей среды;
- неверно указан `host`.

Что делать:

- проверить хост и порт;
- проверить, что есть интернет/маршрут до сервера;
- для локального адреса `192.168.1.6` запускать из вашей сети.

### JDBC подключается, но приложение нет

Проверьте:

- заполнен ли `data/db.properties`;
- скачан ли `lib/postgresql-42.7.4.jar`;
- совпадает ли `db.type` с реальной БД;
- нужен ли `sslMode=require` для PostgreSQL.

### Для другой БД нужны другие запросы

Сейчас в коде полностью поддержаны:

- SQLite
- PostgreSQL
- MySQL
- Firebird / Ред БД

Для Firebird / Ред БД дополнительно нужен JDBC-драйвер `Jaybird` в папке `lib/`.

Для Oracle, SQL Server или другой СУБД дополнительно понадобятся:

- JDBC-драйвер этой БД;
- новый JDBC URL в `DatabaseConfig.java`;
- адаптация SQL в `DatabaseManager.java`;
- адаптация probe-запросов в `DatabaseCli.java`.

Причина в том, что текущий код использует SQL-конструкции, совместимые именно с SQLite/PostgreSQL:

- `AUTOINCREMENT` / `BIGSERIAL` / `AUTO_INCREMENT` / `IDENTITY`
- `ON CONFLICT` / `ON DUPLICATE KEY UPDATE` / `UPDATE OR INSERT`
- `PRAGMA`
- `LIMIT`

У другой СУБД эти места могут отличаться.
