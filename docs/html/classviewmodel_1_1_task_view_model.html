Подробное описание

ViewModel для отображения задачи в пользовательском интерфейсе

Реализует паттерн ViewModel в архитектуре MVVM. Служит промежуточным слоем между моделью данных (Task) и представлением (JavaFX UI). Предоставляет JavaFX свойства для двусторонней привязки данных и форматирует данные для удобного отображения в таблице.

Автор
Чернов
Версия
1.0
Дата
2025-11-4
Заметки
Все свойства являются Observable, что позволяет автоматически обновлять UI при изменении данных и наоборот.

◆ TaskViewModel()

viewmodel.TaskViewModel.TaskViewModel ( Task task)

Конструктор на основе модели Task.

Аргументы
taskИсходный объект задачи

Создает ViewModel, инициализируя все JavaFX свойства значениями из переданного объекта Task. Устанавливает двусторонние привязки между свойствами ViewModel и полями модели.

Аргументы
taskОбъект модели Task, не может быть null
Исключения
NullPointerExceptionесли task равен null
Заметки
После создания ViewModel изменения в свойствах автоматически синхронизируются с исходной моделью Task.

◆ setupBindings()

void viewmodel.TaskViewModel.setupBindings ( )
private

Настраивает двусторонние привязки между ViewModel и Model.

Устанавливает слушатели изменений для всех свойств ViewModel. При изменении любого свойства в UI (например, редактирование в таблице) соответствующее изменение применяется к исходной модели Task.

Заметки
Привязки обеспечивают автоматическую синхронизацию данных между представлением и моделью без необходимости ручного обновления.
Предупреждения
Изменение поля id не поддерживается, так как ID неизменяем
См. также
javafx.beans.property.Property#addListener

◆ updateFromModel()

void viewmodel.TaskViewModel.updateFromModel ( )

Обновляет ViewModel на основе изменений в модели

Синхронизирует все свойства ViewModel с текущими значениями из связанного объекта Task. Используется при внешних изменениях модели, которые не были инициированы через UI.

Заметки
Этот метод следует вызывать при изменении модели Task извне, например, при загрузке задач из файла или изменении через TaskManager.
